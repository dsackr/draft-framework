# DRAFT Framework History

DRAFT — Deployable Reference Architecture Framework Toolkit

## Origins

DRAFT was conceived and developed by Dale Sackrider as a practical alternative
to documentation-heavy enterprise architecture frameworks. It is loosely inspired
by TOGAF, but it intentionally favors rules-based validation, reusable object
types, and Git-as-source-of-truth over presentation-heavy governance processes.

The core principle has remained consistent across every edition: architecture
should be expressed as deployable, machine-readable artifacts rather than slide
decks, static diagrams, or disconnected prose documents.

## Founding

The framework was first implemented through hands-on architecture interview and
platform standardization work. From the beginning, the goal was to create a
toolkit that architects and engineers could actually use while designing,
reviewing, and evolving real deployments.
